2. MarkDown 문법

January 7, 2024 · Kim-Minjoon

0. Markdown이란#

github에 들어가본 사람이라면 한번쯤 .md 확장자 파일을 본 적 있을 것이다. markdown은 간단한 문법으로 쉽게 쓰고 읽을 수 있다. 다음과 같은 장단점이 존재한다.

장점

  • 간결하다
  • 텍스트 에디터로 쉽게 작성 가능하다.
  • 다양한 프로그램과 플랫폼에서 활용할 수 있다.

단점

  • 표준이 없다.
  • 모든 HTML을 표기할 수 없다.

1. 문법#

  • Header

다음과 같이 제목을 표기할 수 있다.

1
2
3

    # Header 1
    ## Header 2
    ### Header 3

Header 1#

Header 2#

Header 3#

  • BlockQuote
1
2
3

>first
> > second
> > > third

first

second

third

  • List
1
2
3

1. first
2. second
3. third
  1. first
  2. second
  3. third

순서를 바꾸어도 순서’만’ 정렬된다.

1
2
3

1. first
3. third
2. second
  1. first
  2. third
  3. second
  • 순서없는 list (*, +, -)
1
2
3

* first
    * second
        * third
  • first
    • second
      • third
  • code

다음과 같이 코드 들여쓰기를 할 수 있다.

1
2
3

    ```c
    printf("hello world\n");
    ```

1

    printf("hello world\n");
  • vertical bar
1
2

---
***
  • link
1
2

    [Title](link)
    [Google](https://google.com)

Google
